    ```python
    class ProjectResolver:
        """Resolves which Linear project/team an intent should route to.

        Resolution priority:
        1. Explicit project name in the intent (LLM extracted "ACRE" from transcript)
        2. Active conversation topic (sustained mention tracking)
        3. Attendee's primary company -> matching Linear team
        4. Default team (config.LINEAR_DEFAULT_TEAM_ID or first available)
        """

        def __init__(self):
            self._project_cache: dict[str, LinearProject] = {}  # name_lower -> project
            self._cache_expiry: float = 0
            self._cache_ttl: float = 300  # 5 minutes
            self._active_project: LinearProject | None = None
            self._topic_window: list[str] = []  # last N project mentions for switching detection
            self._topic_threshold: int = 3  # consecutive mentions before switching
            self.logger = logging.getLogger("copilot.routing")
    ```

    Methods:

    1. `async def refresh_cache(self) -> None` — Fetch all teams from Linear via GraphQL (query `teams { nodes { id name key description } }`). Populate `_project_cache` keyed by lowercase name. Only refresh if cache expired.

    2. `async def resolve(self, intent: Intent, attendee_companies: list[str] | None = None) -> LinearProject | None`:
       - If `intent.project` is set, search cache for fuzzy match (containsIgnoreCase). If not in cache, do a fresh Linear API lookup via `fetch_linear_projects(intent.project)`. Return first match.
       - If no explicit project, check `_active_project` (set by topic switching).
       - If no active project, try each company in `attendee_companies` against cache.
       - Return None if no match found (caller decides: create new or use default).

    3. `def track_topic(self, project_name: str | None) -> LinearProject | None`:
       - If project_name is None, clear the window.
       - Append project_name to `_topic_window` (keep last 10).
       - Count last `_topic_threshold` entries. If all same project, switch `_active_project`.
       - Return new active project if switched, None if no change.
       - Log at INFO when switching: "Topic switch detected: {old} -> {new}"

    4. `async def find_or_create_team(self, name: str) -> LinearProject`:
       - First try `resolve(Intent(project=name, ...))` to find existing.
       - If not found, create new team via Linear GraphQL mutation:
         ```graphql
         mutation CreateTeam($input: TeamCreateInput!) {
             teamCreate(input: $input) {
                 success
                 team { id name key }
             }
         }
         ```
         Input: `name=name`, `key=` first 4 uppercase chars of name (alphanumeric only).
       - Add to cache, return as LinearProject.
       - Log at INFO: "Created new Linear team: {name} ({key})"

    Helper: `async def _graphql(self, query: str, variables: dict | None = None) -> dict` — same pattern as linear_client.py `_graphql_request` but as an instance method using httpx, reading `config.LINEAR_API_KEY`.

    **IMPORTANT:** Use `IssueCreateInput` mutation variables (not string interpolation) for all GraphQL mutations per research anti-patterns.

    Add `if __name__ == "__main__"` block for CLI testing that lists all cached projects.

    Create `engine/routing/linear_router.py` with the `LinearRouter` class.

    ```python
    class LinearRouter:
        """Creates Linear issues routed to the correct project/team."""

        def __init__(self, resolver: ProjectResolver | None = None):
            self.resolver = resolver or ProjectResolver()
            self.logger = logging.getLogger("copilot.routing.linear")
    ```

    Methods:

    1. `async def route_intent(self, intent: Intent, attendee_companies: list[str] | None = None, meeting_title: str | None = None) -> dict | None`:
       - Resolve project via `self.resolver.resolve(intent, attendee_companies)`.
       - If no project found and `intent.project` is set, auto-create via `self.resolver.find_or_create_team(intent.project)`.
       - If still no project, use `config.LINEAR_DEFAULT_TEAM_ID` (add this to config.py — default empty string; if empty, log warning and return None).
       - Call `create_linear_issue(intent, team_id, meeting_title)`.
       - Track topic: `self.resolver.track_topic(project.name if project else None)`.
       - Return the created issue dict (id, identifier, url) or None on failure.

    2. `async def create_linear_issue(self, intent: Intent, team_id: str, meeting_title: str | None = None) -> dict | None`:
       - Build description from intent context:
         ```
         **From meeting:** {meeting_title or "Unknown meeting"}
         **Speaker:** {intent.speaker}
         **Urgency:** {intent.urgency}
         **Confidence:** {intent.confidence:.0%}

         **Context:**
         > {intent.source_text}

         **Details:**
         {intent.details}

         _Created automatically by Meeting Copilot_
         ```
       - Map intent to issue title: Use `intent.target` as title, prefixed by action hint:
         - BUILD_FEATURE -> "Build: {target}"
         - FIX_BUG -> "Fix: {target}"
         - RESEARCH -> "Research: {target}"
         - FOLLOW_UP -> "Follow up: {target}"
         - SEND_EMAIL -> "Email: {target}"
         - Others -> just `intent.target`
       - Map urgency to Linear priority: "now" -> 1 (urgent), "soon" -> 3 (medium), "later" -> 4 (low)
       - GraphQL mutation using variables (NOT string interpolation):
         ```graphql
         mutation CreateIssue($input: IssueCreateInput!) {
             issueCreate(input: $input) {
                 success
                 issue { id identifier title url state { name } }
             }
         }
         ```
         Variables: `{"input": {"title": title, "description": description, "teamId": team_id, "priority": priority}}`
       - Return issue dict on success, None on failure.
       - Log at INFO: "Created Linear issue: {identifier} in {team_key} -> {url}"

    3. `async def route_batch(self, intents: list[Intent], attendee_companies: list[str] | None = None, meeting_title: str | None = None) -> list[dict]`:
       - Route each intent, return list of created issues (skipping None results).
       - Use `asyncio.gather` for parallel creation.
