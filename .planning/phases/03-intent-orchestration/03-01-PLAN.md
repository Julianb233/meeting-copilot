    ```python
    class ActionType(str, Enum):
        CREATE_ISSUE = "create_issue"
        BUILD_FEATURE = "build_feature"
        FIX_BUG = "fix_bug"
        RESEARCH = "research"
        SEND_EMAIL = "send_email"
        CREATE_PROPOSAL = "create_proposal"
        SCHEDULE_MEETING = "schedule_meeting"
        CHECK_DOMAIN = "check_domain"
        DEPLOY = "deploy"
        DECISION = "decision"
        FOLLOW_UP = "follow_up"
        GENERAL_TASK = "general_task"

    class Intent(BaseModel):
        action_type: ActionType
        target: str                         # What to act on ("login page", "landing page")
        urgency: Literal["now", "soon", "later"] = "soon"
        project: str | None = None          # Detected project name from conversation
        assignee: str | None = None
        details: str                        # Full context extracted
        confidence: float                   # 0.0-1.0
        source_text: str                    # Original transcript text
        speaker: str                        # Who said it
        requires_agent: bool = False        # Whether this needs fleet agent execution

    class ClassifiedSentence(BaseModel):
        index: int
        text: str
        speaker: str
        classification: str                 # INFO, ACTION_ITEM, DECISION, FOLLOW_UP, QUESTION

    class IntentBatch(BaseModel):
        intents: list[Intent] = Field(default_factory=list)
        classifications: list[ClassifiedSentence] = Field(default_factory=list)
        model_used: str = "keywords"
        processing_time_ms: float = 0.0
    ```

    **engine/intent/prompts.py:**
    Two prompt templates:

    1. `CLASSIFY_BATCH_PROMPT` — Adapted from meeting-watcher's `CLASSIFY_PROMPT`. Same 5 categories (ACTION_ITEM, DECISION, FOLLOW_UP, QUESTION, INFO). Returns JSON array of `{"index": N, "classification": "..."}`. Format function: `format_classify_prompt(sentences: list[dict]) -> str`.

    2. `EXTRACT_INTENTS_PROMPT` — New structured extraction prompt. Takes actionable sentences + meeting context (attendee names, known projects). Returns JSON array of Intent-shaped objects. Include in the prompt:

       - Urgency heuristic: "now" if speaker says immediately/right now/ASAP, "later" if explicitly deferred, "soon" otherwise
       - `requires_agent: true` for BUILD_FEATURE, FIX_BUG, RESEARCH, DEPLOY, CHECK_DOMAIN, CREATE_PROPOSAL
       - `requires_agent: false` for CREATE_ISSUE, DECISION, FOLLOW_UP, SEND_EMAIL, SCHEDULE_MEETING, GENERAL_TASK
       - Project detection hint: list known projects from context so LLM can match

       Format function: `format_extract_prompt(sentences: list[dict], known_projects: list[str], attendee_names: list[str]) -> str`.

    3. `classify_keywords(text: str) -> str` — Port the keyword classifier from meeting-watcher.py lines 335-361 exactly. Same keyword lists for ACTION_ITEM, DECISION, FOLLOW_UP, QUESTION.

    **engine/intent/fallback_chain.py:**
    Implement the FallbackChain pattern from 03-RESEARCH.md Pattern 1:

    - `ModelHealth` dataclass: `last_fail`, `consecutive_fails`, `cooldown_seconds=300`. Methods: `is_available()`, `record_failure()`, `record_success()`.
    - `ModelProvider` dataclass: `name`, `call` (async callable), `health`.
    - `FallbackChain` class with `providers: list[ModelProvider]`.
      - `async def call(self, prompt: str) -> tuple[str | None, str]` — tries each provider in order, returns (response_text, model_name). Returns (None, "none") if all fail.

    - Three async provider functions (standalone, not methods):
      - `async def call_gemini(prompt: str) -> str` — Use httpx async POST to `https://generativelanguage.googleapis.com/v1beta/models/gemini-2.0-flash:generateContent?key={key}`. Body: `{"contents": [{"parts": [{"text": prompt}]}]}`. Parse: `resp["candidates"][0]["content"]["parts"][0]["text"]`. Timeout 30s.
      - `async def call_anthropic(prompt: str) -> str` — httpx POST to `https://api.anthropic.com/v1/messages`. Model: `claude-haiku-4-5-20251001`. Headers: `x-api-key`, `anthropic-version: 2023-06-01`. Max tokens 1024.
      - `async def call_openai(prompt: str) -> str` — httpx POST to `https://api.openai.com/v1/chat/completions`. Model: `gpt-4o-mini`. Max tokens 1024.

    - Factory function `create_default_chain() -> FallbackChain` that builds the chain in order: Gemini (if GEMINI_API_KEY set) -> Anthropic (if ANTHROPIC_API_KEY set) -> OpenAI (if OPENAI_API_KEY set). NOTE: Gemini FIRST because Anthropic has zero credits currently.

    **IMPORTANT:** All API calls via httpx (async), NOT urllib. Strip markdown code fences from responses before returning (regex: strip ```json and ``` wrappers).

    Create `engine/intent/detector.py` with the `IntentDetector` class implementing the two-stage pipeline from 03-RESEARCH.md Pattern 3.

    ```python
    class IntentDetector:
        def __init__(self, chain: FallbackChain | None = None):
            self.chain = chain or create_default_chain()
            self.logger = logging.getLogger("copilot.intent")

        async def process_sentences(
            self,
            sentences: list[dict],  # Each dict has "text", "speaker" (optionally "speaker_name")
            known_projects: list[str] | None = None,
            attendee_names: list[str] | None = None,
        ) -> IntentBatch:
            """Two-stage intent detection pipeline.

            Stage 1: Batch classify sentences (INFO/ACTION_ITEM/DECISION/FOLLOW_UP/QUESTION)
            Stage 2: Extract structured intents from non-INFO sentences only
            """
            start = time.monotonic()
            known_projects = known_projects or []
            attendee_names = attendee_names or []

            # Stage 1: Batch classification
            classifications = await self._classify_batch(sentences)

            # Stage 2: Extract intents from actionable sentences
            actionable = [
                (s, c) for s, c in zip(sentences, classifications)
                if c.classification != "INFO"
            ]

            intents: list[Intent] = []
            model_used = "keywords"

            if actionable:
                intents, model_used = await self._extract_intents(
                    [s for s, _ in actionable],
                    known_projects,
                    attendee_names,
                )

            elapsed = (time.monotonic() - start) * 1000
            self.logger.info(
                "Processed %d sentences -> %d actionable -> %d intents (%.0fms, model=%s)",
                len(sentences), len(actionable), len(intents), elapsed, model_used,
            )

            return IntentBatch(
                intents=intents,
                classifications=classifications,
                model_used=model_used,
                processing_time_ms=elapsed,
            )
    ```

    **_classify_batch method:**
    1. Format sentences with `format_classify_prompt(sentences)`.
    2. Call `self.chain.call(prompt)` to get `(response_text, model_name)`.
    3. If response_text is None, fall back to `classify_keywords` for each sentence.
    4. If response_text is not None, parse JSON array. Handle: JSON wrapped in markdown code fences (strip them). Handle: malformed JSON (fallback to keywords).
    5. Build list of `ClassifiedSentence` for each input sentence. If a sentence wasn't in the LLM response, default to "INFO".

    **_extract_intents method:**
    1. Format prompt with `format_extract_prompt(sentences, known_projects, attendee_names)`.
    2. Call `self.chain.call(prompt)`.
    3. If response is None, create basic Intent objects from the classified sentences using heuristics:
       - ACTION_ITEM -> `ActionType.GENERAL_TASK`, confidence=0.5
       - DECISION -> `ActionType.DECISION`, confidence=0.5
       - FOLLOW_UP -> `ActionType.FOLLOW_UP`, confidence=0.5
    4. If response is not None, parse JSON array into Intent objects using `Intent.model_validate()` with try/except for each item (skip malformed).
    5. Return `(intents, model_name)`.

    **JSON parsing helper `_parse_json_response(text: str) -> list[dict] | None`:**
    - Strip markdown code fences (```json ... ``` or ``` ... ```).
    - Try `json.loads(text)`.
    - If fails, try regex `re.search(r'\[.*\]', text, re.DOTALL)` and parse that.
    - Return None if all parsing fails.
