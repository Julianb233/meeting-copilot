    ```python
    class TaskState(str, Enum):
        PENDING = "pending"
        RUNNING = "running"
        COMPLETED = "completed"
        FAILED = "failed"

    class TrackedTask(BaseModel):
        id: str                          # UUID
        intent_action: str               # ActionType value
        target: str                      # What the task acts on
        agent: str | None = None         # Which fleet agent is assigned
        state: TaskState = TaskState.PENDING
        created_at: datetime = Field(default_factory=lambda: datetime.now(timezone.utc))
        started_at: datetime | None = None
        completed_at: datetime | None = None
        result: str | None = None        # Output or error message
        linear_issue_id: str | None = None  # Linked Linear issue if created
        process_pid: int | None = None   # PID of spawned subprocess

    class TaskTracker:
        """In-memory task lifecycle tracker.

        Keeps all tasks from the current meeting session. Provides
        snapshot for WebSocket broadcast and status queries.
        """

        def __init__(self, max_history: int = 50):
            self._tasks: dict[str, TrackedTask] = {}  # id -> task
            self._max_history = max_history
            self.logger = logging.getLogger("copilot.orchestration.tracker")
    ```

    Methods:

    1. `def create_task(self, intent: Intent, agent: str | None = None) -> TrackedTask`:
       - Generate UUID for task id.
       - Create TrackedTask from intent fields.
       - Store in `_tasks`.
       - Trim oldest completed/failed tasks if exceeding `_max_history`.
       - Return the task.

    2. `def start_task(self, task_id: str, agent: str, pid: int | None = None) -> TrackedTask | None`:
       - Set state to RUNNING, assign agent, set started_at, optionally set process_pid.
       - Return updated task or None if not found.

    3. `def complete_task(self, task_id: str, result: str) -> TrackedTask | None`:
       - Set state to COMPLETED, set result, set completed_at.

    4. `def fail_task(self, task_id: str, error: str) -> TrackedTask | None`:
       - Set state to FAILED, set result to error message, set completed_at.

    5. `def get_task(self, task_id: str) -> TrackedTask | None`.

    6. `def get_active_tasks(self) -> list[TrackedTask]`:
       - Return tasks with state PENDING or RUNNING.

    7. `def get_all_tasks(self) -> list[TrackedTask]`:
       - Return all tasks sorted by created_at descending.

    8. `def get_agent_status(self) -> list[dict]`:
       - Return list of `{"name": agent_name, "status": "busy"|"idle", "current_task": task_target|None}` for agents 1-4.
       - An agent is "busy" if it has a RUNNING task.

    9. `def snapshot(self) -> dict`:
       - Return `{"active": [...], "completed": [...], "agents": [...]}` for WebSocket broadcast.
  </action>

    from intent.models import Intent, ActionType

    Create `engine/orchestration/fleet_spawner.py` with the `FleetSpawner` class.

    ```python
    # Agent specialization map — ported from fleet task-router.js concept
    # Maps action types to preferred agents
    AGENT_SPECIALIZATIONS: dict[str, list[str]] = {
        "build_feature": ["agent1", "agent2", "agent3"],
        "fix_bug": ["agent1", "agent2"],
        "research": ["agent3", "agent4"],
        "deploy": ["agent1"],
        "check_domain": ["agent4"],
        "create_proposal": ["agent4", "agent3"],
        "send_email": ["agent4"],
    }

    GOD_BIN = "/usr/local/bin/god"

    class FleetSpawner:
        """Spawns fleet agents to execute meeting intents.

        Uses `god` CLI via async subprocess. Agent selection
        based on action type specialization and current availability.
        """

        def __init__(self, tracker: TaskTracker):
            self.tracker = tracker
            self.logger = logging.getLogger("copilot.orchestration.spawner")
    ```

    Methods:

    1. `def select_agent(self, intent: Intent) -> str`:
       - Look up `AGENT_SPECIALIZATIONS[intent.action_type.value]`.
       - Get current agent status from `self.tracker.get_agent_status()`.
       - Prefer idle agents from the specialization list.
       - If all specialized agents busy, pick any idle agent.
       - If all agents busy, return the first specialized agent (will queue).
       - Return agent name (e.g., "agent1").

    2. `async def spawn(self, intent: Intent, meeting_title: str | None = None) -> TrackedTask`:
       - Create task via tracker: `task = self.tracker.create_task(intent)`.
       - Select agent: `agent = self.select_agent(intent)`.
       - Build the task prompt for the agent. This is what gets passed to the agent's Claude instance:
         ```
         Task from meeting: {meeting_title or "Active meeting"}
         Action: {intent.action_type.value}
         Target: {intent.target}
         Details: {intent.details}
         Urgency: {intent.urgency}
         Source: "{intent.source_text}" — {intent.speaker}
         ```
       - Spawn via async subprocess. The command depends on action_type:
         - For BUILD_FEATURE, FIX_BUG, DEPLOY: Use `god mac exec` to send a task (since agents run as PM2 processes, we use iMessage to dispatch for now). Command: `god mac send "+16195090699" "Fleet task for {agent}: {prompt_summary}"`. This is the v1 approach — proper fleet gateway integration comes in Phase 6.
         - For RESEARCH, CHECK_DOMAIN: Use subprocess directly: `asyncio.create_subprocess_exec(GOD_BIN, "mac", "exec", f"curl -s 'https://dns.google/resolve?name={target}&type=A'" ...)` for domain checks, or a simple research prompt.
         - For all types, the general pattern is:
           ```python
           proc = await asyncio.create_subprocess_exec(
               GOD_BIN, "mac", "send", "+16195090699",
               f"[Meeting Copilot] Task for {agent}: {intent.action_type.value} - {intent.target}\n{intent.details}",
               stdout=asyncio.subprocess.PIPE,
               stderr=asyncio.subprocess.PIPE,
           )
           ```
       - Start task in tracker: `self.tracker.start_task(task.id, agent, pid=proc.pid)`.
       - Fire-and-forget: spawn a background asyncio task to wait for process completion:
         ```python
         asyncio.create_task(self._wait_for_completion(task.id, proc))
         ```
       - Return the TrackedTask immediately (non-blocking).

    3. `async def _wait_for_completion(self, task_id: str, proc: asyncio.subprocess.Process) -> None`:
       - `stdout, stderr = await asyncio.wait_for(proc.communicate(), timeout=300)` (5 min timeout).
       - If returncode 0: `self.tracker.complete_task(task_id, stdout.decode().strip() or "Task dispatched")`.
       - If returncode != 0 or timeout: `self.tracker.fail_task(task_id, stderr.decode().strip() or "Process failed")`.
       - Log result.

    4. `async def spawn_batch(self, intents: list[Intent], meeting_title: str | None = None) -> list[TrackedTask]`:
       - Filter to `requires_agent=True` intents only.
       - Spawn each, return list of TrackedTasks.

    Add `if __name__ == "__main__"` block that creates a test intent and shows the selected agent (without actually spawning).
