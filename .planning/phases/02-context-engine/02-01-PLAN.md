<objective>
Build the shared context models and attendee identity resolution module. Identity resolution looks up attendee emails via Google Contacts (People API) using the `gws` CLI tool, with an Obsidian Contacts fallback.

Purpose: When a meeting starts and attendee emails are known (from Calendar), resolve each email to a full identity (name, company, phone, photo) so the context engine knows WHO is in the meeting. This identity is used by downstream loaders (Linear needs company name, profiles need email+company).

Output: `engine/context/models.py` (shared models for all context modules), `engine/context/contacts.py` (identity resolution).
</objective>

    ```python
    class AttendeeIdentity(BaseModel):
        email: str
        display_name: str | None = None
        given_name: str | None = None
        family_name: str | None = None
        organization: str | None = None
        title: str | None = None
        phone: str | None = None
        photo_url: str | None = None
        source: str = "unknown"  # "google_contacts" | "obsidian" | "calendar" | "email_only"

    class TranscriptSummary(BaseModel):
        id: str
        title: str
        date: datetime | None = None
        duration_minutes: float | None = None
        summary: str | None = None
        action_items: str | None = None
        short_summary: str | None = None
        participants: list[str] = Field(default_factory=list)
        transcript_url: str | None = None

    class LinearIssue(BaseModel):
        id: str
        identifier: str  # e.g. "ACRE-42"
        title: str
        state: str
        priority_label: str | None = None
        assignee: str | None = None
        url: str | None = None

    class LinearProject(BaseModel):
        id: str
        name: str
        slug_id: str | None = None
        state: str | None = None
        description: str | None = None
        open_issues: list[LinearIssue] = Field(default_factory=list)

    class ClientProfile(BaseModel):
        slug: str
        client_name: str
        status: str | None = None
        emails: list[str] = Field(default_factory=list)  # all emails found in profile
        communication_style: str | None = None
        formality: str | None = None
        tone: str | None = None
        tech_understanding: str | None = None
        raw_content: str = ""  # full markdown for classifier prompt context

    class AttendeeContext(BaseModel):
        identity: AttendeeIdentity
        meeting_history: list[TranscriptSummary] = Field(default_factory=list)
        linear_projects: list[LinearProject] = Field(default_factory=list)
        client_profile: ClientProfile | None = None

    class UnifiedMeetingContext(BaseModel):
        meeting_title: str | None = None
        attendees: list[AttendeeContext] = Field(default_factory=list)
        assembled_at: datetime = Field(default_factory=lambda: datetime.now(timezone.utc))
        load_time_seconds: float = 0.0
        errors: list[str] = Field(default_factory=list)

        def to_classifier_prompt(self) -> str:
            """Format context for the LLM classifier."""
            # Build a readable text summary of all attendees for LLM consumption.
            # Include: identity, communication style, recent meetings, open issues.
            sections = [f"# Meeting: {self.meeting_title or 'Untitled'}"]
            for att in self.attendees:
                sections.append(f"\n## Attendee: {att.identity.display_name or att.identity.email}")
                if att.identity.organization:
                    sections.append(f"**Organization:** {att.identity.organization}")
                if att.client_profile:
                    if att.client_profile.communication_style:
                        sections.append(f"**Communication Style:** {att.client_profile.communication_style}")
                    if att.client_profile.formality:
                        sections.append(f"**Formality:** {att.client_profile.formality}")
                if att.meeting_history:
                    sections.append(f"\n### Last {len(att.meeting_history)} meetings:")
                    for mh in att.meeting_history:
                        date_str = mh.date.strftime('%Y-%m-%d') if mh.date else 'unknown date'
                        sections.append(f"- {mh.title} ({date_str}): {mh.short_summary or mh.summary or 'No summary'}")
                if att.linear_projects:
                    for proj in att.linear_projects:
                        if proj.open_issues:
                            sections.append(f"\n### Open issues in {proj.name} ({len(proj.open_issues)}):")
                            for issue in proj.open_issues[:5]:
                                sections.append(f"- [{issue.identifier}] {issue.title} ({issue.state})")
            return "\n".join(sections)
    ```

  <name>Task 2: Create contacts lookup with name-based search and Obsidian fallback</name>

  <action>
    Create `engine/context/contacts.py` with:

    **CRITICAL from research:** Google Contacts `searchContacts` does NOT match email addresses. It only matches names, nicknames, and organizations. You MUST search by display name, not by email.

    **Function `resolve_attendee(email: str, display_name: str | None = None) -> AttendeeIdentity`:**

    Resolution strategy (in order):

    1. **Obsidian Contacts lookup (fastest, local):**
       - Scan `/opt/agency-workspace/obsidian-vault/Contacts/*.md` files
       - Parse YAML frontmatter with `frontmatter.load()`
       - Match by `email` field in frontmatter (case-insensitive)
       - If found, extract: name, email, company, role from frontmatter
       - Set `source="obsidian"`

    2. **Google Contacts lookup (if display_name available OR we got a name from Obsidian):**
       - Use `asyncio.create_subprocess_exec` to call:
         `gws people people searchContacts --params '{"query":"<name_query>","readMask":"names,emailAddresses,organizations,phoneNumbers,photos"}'`
       - The `query` parameter should be the display_name (from calendar attendee data), or the name found from Obsidian, or as last resort the email local part (e.g., "jan.gleisner" from "jan@hafniafin.com" — replace dots with spaces).
       - Parse JSON stdout. Response has `results[].person`.
       - **Confirm match:** Iterate results, find the person whose `emailAddresses[].value` matches the input email (case-insensitive). Do NOT trust the first result — searchContacts does fuzzy name matching.
       - Extract: `names[0].displayName`, `organizations[0].name`, `organizations[0].title`, `phoneNumbers[0].value`, `photos[0].url`
       - Set `source="google_contacts"`
       - 10-second timeout on subprocess. On timeout or error, skip Google and use whatever we found from Obsidian.

    3. **Fallback:** If neither source found the email, return `AttendeeIdentity(email=email, display_name=display_name, source="email_only")`.

    **Merge strategy:** If both Obsidian and Google return data, prefer Google for phone/photo (richer), Obsidian for company/role/name (more reliable for our contacts).

    **Function `resolve_attendees(emails: list[str], display_names: dict[str, str] | None = None) -> list[AttendeeIdentity]`:**
    - `display_names` is an optional dict mapping email -> display name (from calendar event attendee data)
    - Use `asyncio.gather(*[resolve_attendee(e, display_names.get(e) if display_names else None) for e in emails])`
    - Return list in same order as input emails

    **Add `if __name__ == "__main__"` block:**
    - Takes email from sys.argv[1] or defaults to "jan@hafniafin.com"
    - Optionally takes --name "Jan Gleisner" for display name
    - Calls `asyncio.run(resolve_attendee(email, name))`
    - Prints result as JSON (model_dump_json with indent=2)
    - Also tests with unknown email to verify fallback

    Add logging with `logging.getLogger(__name__)`.
