  <name>Task 1: Create AttendeeIdentity model and contacts lookup module</name>
  <files>engine/context/__init__.py, engine/context/contacts.py</files>
  <action>
    1. Create `engine/context/__init__.py` — empty init file to make `context` a package.

    2. Create `engine/context/contacts.py` with:

    **Pydantic model:**
    ```python
    class AttendeeIdentity(BaseModel):
        email: str
        name: str | None = None
        company: str | None = None
        title: str | None = None
        phone: str | None = None
        photo_url: str | None = None
        source: str = "google_contacts"  # or "email_only" if not found
    ```

    **Function `resolve_attendee(email: str) -> AttendeeIdentity`:**
    - Use `asyncio.create_subprocess_exec` to call:
      `gws people people searchContacts --params '{"query":"<email>","readMask":"names,emailAddresses,organizations,phoneNumbers,photos"}'`
    - Parse the JSON stdout. The response has `results[].person` with:
      - `names[0].displayName` -> name
      - `organizations[0].name` -> company
      - `organizations[0].title` -> title
      - `phoneNumbers[0].value` -> phone
      - `photos[0].url` -> photo_url
    - Match results: iterate `results`, find the person whose `emailAddresses[].value` matches the query email (case-insensitive). The search query may return fuzzy matches, so confirm the email matches.
    - If no match found or gws returns error, return `AttendeeIdentity(email=email, source="email_only")`.
    - Set a 10-second timeout on the subprocess. On timeout, return email_only fallback.
    - Log the lookup result at INFO level.

    **Function `resolve_attendees(emails: list[str]) -> list[AttendeeIdentity]`:**
    - Use `asyncio.gather(*[resolve_attendee(e) for e in emails])` for parallel resolution.
    - Return list in same order as input emails.

    Important: Do NOT use google-api-python-client or OAuth tokens. The `gws` CLI tool at `/usr/local/bin/gws` already handles auth. Shell out to it.
  </action>
  <verify>
    - `python -c "import engine.context.contacts"` succeeds (run from project root)
    - Or `cd engine && python -c "from context.contacts import resolve_attendee, resolve_attendees, AttendeeIdentity; print('OK')"`
  </verify>
  <done>

    - `resolve_attendee` returns email_only fallback for unknown emails without crashing
    - `resolve_attendees` handles a list of emails in parallel
  </done>
