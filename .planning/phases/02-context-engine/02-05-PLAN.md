  <action>
    Create `engine/context/assembler.py` with:

    **Pydantic models:**
    ```python
    class AttendeeContext(BaseModel):
        """All known context about a single meeting attendee."""
        identity: AttendeeIdentity
        meeting_history: list[TranscriptSummary] = Field(default_factory=list)
        linear_projects: list[LinearProject] = Field(default_factory=list)
        client_profile: ClientProfile | None = None

    class UnifiedMeetingContext(BaseModel):
        """Complete context for a meeting, assembled from all sources."""
        meeting_title: str | None = None
        attendees: list[AttendeeContext] = Field(default_factory=list)
        assembled_at: datetime = Field(default_factory=lambda: datetime.now(timezone.utc))
        load_time_seconds: float = 0.0
        errors: list[str] = Field(default_factory=list)  # track partial failures
    ```

    **Function `assemble_meeting_context(emails: list[str], meeting_title: str | None = None) -> UnifiedMeetingContext`:**

    1. Start timer.

    2. Step 1 — Resolve identities (parallel for all emails):
       ```python
       identities = await resolve_attendees(emails)
       ```

    3. Step 2 — For each attendee, load history + projects + profile in parallel:
       For each (email, identity) pair, create a coroutine that runs:
       - `fetch_meeting_history(email)` — returns TranscriptSummary list
       - `fetch_linear_projects(identity.company)` if identity.company is not None, else skip (return [])
       - `load_client_profile(email=email, company=identity.company)` — returns ClientProfile | None

       Use `asyncio.gather` with `return_exceptions=True` to run ALL attendees' loaders in parallel. This means for 3 attendees, you have up to 9 concurrent operations (3 per attendee x 3 loader types), plus the initial identity resolution.

    4. Assemble AttendeeContext for each attendee from the gathered results.

    5. For any loader that raised an exception, log the error and add to `errors` list, but continue with partial data.

    6. Record load_time_seconds.

    7. Log at INFO: "Context assembled for {n} attendees in {time:.1f}s ({m} errors)"

    **Key design decisions:**
    - Use `asyncio.gather(*tasks, return_exceptions=True)` so one failing loader doesn't block others.
    - Identity resolution MUST complete before project/profile lookups (need company name from identity).
    - History lookup does NOT depend on identity (uses email directly), so it CAN run in parallel with identity resolution. Optimize: start history fetches immediately alongside identity resolution, then use identity results for project/profile lookups.

    **Optimized flow:**
    ```
    Phase A (parallel): resolve_attendees(emails) + [fetch_meeting_history(e) for e in emails]
    Phase B (parallel, after A): [fetch_linear_projects(company) + load_client_profile(email, company) for each attendee]
    ```

    **Add `if __name__ == "__main__"` block:**
    - Takes emails from sys.argv (space-separated) or defaults to ["brendan@acrepartner.com"]
    - Calls `asyncio.run(assemble_meeting_context(emails))`
    - Prints result as JSON with indent=2
