  <action>
    Create `engine/context/fireflies.py` with:

    **Pydantic models:**
    ```python
    class TranscriptSummary(BaseModel):
        id: str
        title: str
        date: datetime | None = None
        duration: float | None = None  # minutes
        summary: str | None = None
        action_items: list[str] = Field(default_factory=list)
        participants: list[str] = Field(default_factory=list)
        transcript_url: str | None = None
    ```

    **Function `fetch_meeting_history(email: str, limit: int = 3) -> list[TranscriptSummary]`:**

    1. Check `config.FIREFLIES_API_KEY` — if empty, log warning and return [].

    2. Use httpx.AsyncClient to POST to `https://api.fireflies.dev/graphql` with:
       - Header: `Authorization: Bearer {FIREFLIES_API_KEY}`
       - Header: `Content-Type: application/json`

    3. GraphQL query to find transcripts by participant email:
       ```graphql
       query {
         transcripts(limit: <limit>) {
           id
           title
           date
           duration
           summary {
             overview
             action_items
           }
           participants
           transcript_url
         }
       }
       ```
       NOTE: Fireflies `transcripts` query does NOT support filtering by participant in the query itself. You must:
       - Fetch recent transcripts (e.g., limit: 20)
       - Filter client-side: keep only those where `participants` list contains the email (case-insensitive match on email portion)
       - Take the first `limit` matches

    4. Parse response. Fireflies returns `data.transcripts[]` array.
       - `date` is a Unix timestamp (milliseconds) — convert to datetime
       - `duration` is in seconds — convert to minutes
       - `summary.overview` -> summary field
       - `summary.action_items` -> action_items list (may be a string, split by newline if needed)
       - `participants` is a list of strings (names or emails)

    5. Error handling:
       - httpx.TimeoutException: log error, return []
       - GraphQL errors (response has `errors` key): log error, return []
       - Any other exception: log error, return []
       - Set timeout=15 on the httpx client

    6. Log at INFO: "Found {n} transcripts for {email}" or "No transcripts found for {email}"

    **Add `if __name__ == "__main__"` block:**
    - Takes email from sys.argv[1] or defaults to a test email
    - Calls `asyncio.run(fetch_meeting_history(email))`
    - Prints results as JSON

    Important: Use `httpx.AsyncClient` (not requests). Import config for the API key. The engine already has httpx in requirements.txt.
