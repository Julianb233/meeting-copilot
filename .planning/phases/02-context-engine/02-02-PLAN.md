  <action>
    Create `engine/context/fireflies.py` with:

    **Import the TranscriptSummary model** from `context.models` (created by plan 02-01). If running before 02-01, the model definition is in the research doc — create a local copy, but prefer importing from `context.models`.

    **CRITICAL from research:** The Fireflies API URL is `https://api.fireflies.ai/graphql` (NOT `api.fireflies.dev`).

    **CRITICAL from research:** The `participants` filter in the transcripts query WORKS. Use it directly — do NOT fetch all transcripts and filter client-side. The verified query is:
    ```graphql
    query($participants: [String!], $limit: Int) {
        transcripts(participants: $participants, limit: $limit) {
            id title date duration
            organizer_email participants
            meeting_attendees { displayName email name }
            summary { overview action_items short_summary }
        }
    }
    ```
    Variables: `{"participants": [email], "limit": limit}`

    **Function `fetch_meeting_history(email: str, limit: int = 3) -> list[TranscriptSummary]`:**

    1. Check `config.FIREFLIES_API_KEY` — if empty, log warning and return [].

    2. Use `httpx.AsyncClient` to POST to `https://api.fireflies.ai/graphql` with:
       - Header: `Authorization: Bearer {FIREFLIES_API_KEY}`
       - Header: `Content-Type: application/json`
       - Timeout: 15 seconds

    3. Use the verified GraphQL query above with `participants` filter.

    4. Parse response:
       - Check for `errors` key in response JSON — if present, log and return []
       - Extract `data.transcripts` array
       - For each transcript, build a TranscriptSummary:
         - `id` -> id
         - `title` -> title
         - `date` -> **CRITICAL: This is Unix milliseconds.** Convert: `datetime.fromtimestamp(date / 1000, tz=timezone.utc)` — divide by 1000 first!
         - `duration` -> convert seconds to minutes (divide by 60)
         - `summary.overview` -> summary
         - `summary.action_items` -> action_items (may be a string with newlines)
         - `summary.short_summary` -> short_summary
         - `participants` -> participants list

    5. Error handling:
       - httpx.TimeoutException: log error, return []
       - httpx.HTTPStatusError: log error, return []
       - JSON decode error: log error, return []
       - Any other exception: log error, return []

    6. Log at INFO: "Found {n} transcripts for {email}" or "No transcripts found for {email}"

    **Add `if __name__ == "__main__"` block:**
    - Takes email from sys.argv[1] or defaults to "jan@hafniafin.com"
    - Calls `asyncio.run(fetch_meeting_history(email))`
    - Prints results as JSON (using model_dump_json for each)

    Use `logging.getLogger(__name__)` for all logging.
