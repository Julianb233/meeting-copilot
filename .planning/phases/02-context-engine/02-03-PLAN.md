  <action>
    Create `engine/context/linear_client.py` with:

    **Import models** from `context.models` (LinearProject, LinearIssue).

    **CRITICAL from research:** Linear has NO email-to-project mapping. Must build one from client-profiles. The client-profiles directory has markdown files with `slug:` in frontmatter that matches Linear project names. Known mappings:
    - `hafniafin.com` -> slug `hafnia-financial` -> Linear project "hafnia-financial"
    - `acrepartner.com` -> slug `acre-partner` -> Linear project "Acre Partners -- Content Engine"
    - `brandonlehmanbiz@gmail.com` -> slug `brandon-lehman` -> Linear project "Brandon Lehman - AI Detection SaaS"

    **CRITICAL from research:** Linear auth uses bare API key, NO "Bearer" prefix. Header: `Authorization: {LINEAR_API_KEY}`

    **Module-level domain mapping builder:**
    ```python
    def build_domain_mapping() -> dict[str, str]:
        """Build email-domain-to-client-slug mapping from client-profiles directory.
        Returns dict like {"hafniafin.com": "hafnia-financial", "acrepartner.com": "acre-partner"}
        """
    ```
    - Scan `/opt/agency-workspace/client-profiles/*.md` (skip README.md, CLIENT-PORTAL-V2-PLAN.md)
    - Parse YAML frontmatter to get `slug` field
    - Scan content body for email addresses (regex: `\b[\w.-]+@[\w.-]+\.\w+\b`)
    - Map each found email's domain to the slug
    - Also try to map full emails (for gmail.com type addresses where domain is not unique)
    - Cache this at module level (call once at import time or lazily)

    **Helper `_graphql_request(query: str, variables: dict | None = None) -> dict`:**
    - Create `httpx.AsyncClient` (or reuse one)
    - POST to `https://api.linear.app/graphql`
    - Header: `Authorization: {LINEAR_API_KEY}` (bare key, NO "Bearer")
    - Header: `Content-Type: application/json`
    - Body: `{"query": query, "variables": variables or {}}`
    - Timeout: 15 seconds
    - Return parsed JSON response
    - On error: log and raise

    **Function `fetch_linear_projects(company_name: str | None = None, email: str | None = None) -> list[LinearProject]`:**

    1. Check `config.LINEAR_API_KEY` — if empty, log warning and return [].

    2. Determine search term:
       - If email provided, extract domain, look up in domain mapping -> get slug
       - If company_name provided, use it as search term
       - If slug found from domain mapping, use slug as search term
       - If nothing matches, return []

    3. Use `searchProjects` query (verified working in research):
       ```graphql
       query($term: String!, $first: Int) {
           searchProjects(term: $term, first: $first) {
               nodes {
                   id name slugId description state
               }
           }
       }
       ```
       Variables: `{"term": search_term, "first": 5}`

    4. For each matched project, query open issues:
       ```graphql
       query($projectName: String!, $first: Int) {
           issues(
               filter: {
                   project: { name: { containsIgnoreCase: $projectName } }
                   state: { type: { nin: ["completed", "canceled"] } }
               }
               first: $first
           ) {
               nodes {
                   id identifier title
                   state { name type }
                   priority priorityLabel
               }
           }
       }
       ```
       Variables: `{"projectName": project.name, "first": 10}`

    5. Build LinearProject objects with their open issues.

    6. Error handling: httpx.TimeoutException, GraphQL errors, any exception -> log, return [].

    7. Log at INFO: "Found {n} Linear projects for '{search_term}'"

    **Add `if __name__ == "__main__"` block:**
    - Accepts --email or --company from command line
    - Defaults to `--email jan@hafniafin.com`
    - Prints domain mapping, then fetches projects, prints as JSON

    Use `logging.getLogger(__name__)`.
