  <name>Task 1: Deploy panel to Vercel and verify OWASP headers</name>
  <files></files>
  <action>
    Deploy the panel to Vercel:

    1. Run `cd panel && npx vercel --yes` to deploy (creates project if first time)
    2. Capture the deployment URL (e.g., `meeting-copilot-panel-xxx.vercel.app`)
    3. Verify OWASP headers are present:
       ```
       curl -sI https://DEPLOYMENT_URL | grep -iE "(strict-transport|x-content-type|referrer-policy|content-security)"
       ```
       All 4 headers must be present in the response.
    4. Verify the page loads: `curl -s https://DEPLOYMENT_URL | head -20` should return HTML with the React app root div.

    If the Vercel CLI is not authenticated, this will trigger an auth flow -- follow the prompts. If vercel CLI is not installed, run `npm i -g vercel` first.

    Note the deployment URL -- it will be needed for the Zoom Marketplace registration step.
  </action>
  <verify>
    `curl -sI https://DEPLOYMENT_URL` returns all 4 OWASP headers.
    `curl -s https://DEPLOYMENT_URL` returns HTML content.
  </verify>
  <done>Panel is deployed to Vercel with all 4 OWASP headers confirmed in response. Deployment URL is captured.</done>

<task type="checkpoint:human-action" gate="blocking">
  <name>Task 2: Register Zoom App on Marketplace</name>
  <action>
    Julian must register the Meeting Copilot app on the Zoom Marketplace.
  </action>
  <instructions>
    Go to https://marketplace.zoom.us/ and follow these steps:

    1. **Create App:**
       - Click "Develop" > "Build App"
       - Select "General App" > "Create"
       - App name: "Meeting Copilot"
       - Management type: **User-managed**

    2. **Basic Information:**
       - Fill required fields (app name, description, developer contact)
       - No OAuth redirect URL needed (SDK-only approach)

    3. **Features:**
       - Enable "Zoom Apps SDK" feature
       - Under Surface: select "Meeting" (this enables Meeting Side Panel)
       - Client support: Desktop

    4. **Scopes:**
       - Add scope: `zoomapp:inmeeting`

    5. **Surface Configuration:**
       - Home URL: `https://YOUR_VERCEL_DEPLOYMENT_URL` (from Task 1)
       - Domain Allow List: Add these 3 domains:
         - Your Vercel domain (e.g., `meeting-copilot-panel-xxx.vercel.app`)
         - `appssdk.zoom.us`
         - `copilot-api.agency.dev`

    6. **Local Test:**
       - Go to the "Local Test" page
       - Click "Add" to add the app to your Zoom account
       - The app will now appear in Zoom's Apps sidebar

    7. **Verify:**
       - Open Zoom desktop client
       - Join or start a meeting
       - Look for "Meeting Copilot" in the Apps panel/sidebar
       - The panel should render showing the Meeting Copilot UI
  </instructions>
  <resume-signal>Type "registered" with the Vercel deployment URL, or describe any issues encountered.</resume-signal>
</task>
