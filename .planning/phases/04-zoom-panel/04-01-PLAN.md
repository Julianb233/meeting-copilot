Output: vercel.json with headers, useZoomContext hook, updated vite.config.ts with dev headers
</objective>

<task type="auto">
  <name>Task 1: Create vercel.json with OWASP headers and update Vite dev config</name>
  <files>panel/vercel.json, panel/vite.config.ts</files>
  <action>
    Create `panel/vercel.json` with:
    1. Headers block applying to all routes `/(.*)`
    2. All 4 mandatory OWASP headers:
       - `Strict-Transport-Security: max-age=63072000; includeSubDomains`
       - `X-Content-Type-Options: nosniff`
       - `Referrer-Policy: same-origin`
       - `Content-Security-Policy` with:
         - `default-src 'self'`
         - `script-src 'self' 'unsafe-inline' https://appssdk.zoom.us`
         - `style-src 'self' 'unsafe-inline'`
         - `connect-src 'self' wss://copilot-api.agency.dev https://api.zoom.us`
         - `img-src 'self' data:`
         - `font-src 'self' data:`
         - `frame-ancestors 'self'`
    3. Add `rewrites` for SPA routing: `{ "source": "/((?!assets/).*)", "destination": "/index.html" }`

    Update `panel/vite.config.ts`:
    - Add `server.headers` object with the same 4 OWASP headers so local dev mirrors production behavior
    - Keep existing port 5173, react plugin, tailwindcss plugin, build outDir

    Important: Use `copilot-api.agency.dev` as the engine WebSocket domain in CSP connect-src (this is the nginx domain from Phase 1). This must later be mirrored in the Zoom Marketplace Domain Allow List.
  </action>
  <verify>
    Run `cd panel && cat vercel.json | python3 -c "import json,sys; d=json.load(sys.stdin); headers={h['key']:h['value'] for h in d['headers'][0]['headers']}; assert 'Strict-Transport-Security' in headers; assert 'Content-Security-Policy' in headers; print('OK')"` to validate JSON structure.
    Run `cd panel && npx tsc --noEmit` to verify vite.config.ts compiles.
  </verify>
  <done>vercel.json contains all 4 OWASP headers with correct CSP. vite.config.ts has matching dev headers. Both files parse without errors.</done>
</task>

<task type="auto">
  <name>Task 2: Extract useZoomContext hook from App.tsx</name>
  <files>panel/src/hooks/useZoomContext.ts</files>
  <action>
    Create `panel/src/hooks/useZoomContext.ts` that encapsulates Zoom SDK initialization:

    The hook should:
    1. Export a `useZoomContext()` hook returning `{ zoomStatus, meetingContext, isInMeeting }`
    2. `zoomStatus` is typed as `'connecting' | 'connected' | 'error' | 'standalone'`
    3. On mount (useEffect), call `zoomSdk.config()` with capabilities: `getMeetingContext`, `getUserContext`, `getRunningContext`, `expandApp`, `onMeeting`, `openUrl`
    4. On success, set status to `'connected'`
    5. After config, call `zoomSdk.getRunningContext()` -- only call `getMeetingContext()` if `context === 'inMeeting'`
    6. On error (not in Zoom), set status to `'standalone'`
    7. `isInMeeting` derived boolean: true when zoomStatus is 'connected' AND meetingContext is not null
    8. meetingContext should be typed as `{ meetingTopic?: string; meetingID?: string } | null`

    This is extracted from the existing App.tsx useEffect logic (lines 14-35) but improved with:
    - Running context check before getMeetingContext (avoids errors outside meetings)
    - getUserContext and expandApp capabilities added
    - Proper TypeScript return type
    - isInMeeting convenience boolean

    Do NOT modify App.tsx in this task -- Plan 02 handles the full App.tsx refactor.
  </action>
  <verify>Run `cd panel && npx tsc --noEmit` to verify the hook compiles with no type errors.</verify>
  <done>useZoomContext hook exists, exports correctly, compiles without errors, encapsulates all Zoom SDK initialization logic with running context detection.</done>
